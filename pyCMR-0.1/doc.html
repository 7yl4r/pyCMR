pyCMR

Currenlty the library provides separate apis for collections/ granules searching and download is available for all the returned results.

Example usage

>>> import pyCMR
>>> Search collection with specific id
>>> results = pyCMR.searchCollection(limit=50, concept_id="C90757596-LAADS")
==== Waiting for response ====
>>> len(results)
1
>>> results
[{u'organizations': [u'NASA/GSFC/SED/ESD/HBSL/BISB/LAADS', u'NASA/GSFC/SED/ESD/BSL/MCST', u'NASA/GSFC/SED/ESD/HBSL/BISB/MODAPS'], ...... ]
>>> 
>>> # Simply access the result as a python dict
>>> results[0].keys()
[u'organizations', u'updated', u'orbit_parameters', u'time_start', u'links', u'short_name', u'title', u'coordinate_system', u'boxes', u'version_id', u'summary', u'browse_flag', u'collection_data_type', u'dif_ids', u'data_center', u'id', u'dataset_id', u'on
line_access_flag']
>>> results[0]['dataset_id']
u'MODIS/Terra Calibrated Radiances 5-Min L1B Swath 500m V005'
>>> # Download a result to current directory
>>> results[0].download()
>>>
>>> # Download a result to a user defined directory
>>> results[0].download('.../path/to/your/directory')
>>>
>>> # Get the first 50 records of granule
>>> # Note: This api can be extremly slow when limit is large!!!
>>> results = cmr.searchGranule(limit=50)
>>>
>>> # Download all the granules
>>> for res in results:
>>>     res.download('./downloads')

Api Details

The two apis for seaching collections and granules built on top of the CMR Api:

  • searchGranule(limit=100, *args, **kwargs)

  • searchCollection(limit=100, *args, **kwargs)

    Return a list of Result instance (as describes below)

    Input parameters:

    • limit: The limit of result items. Note: Since the CMR api has a limit on the number of results of each query, it would be time consuming for this method to return if the limit is too large. (it is 100 by default)
    • **kwargs: Use exactly the same parameter names as the CMR apis. For instance, if the cmr api describes as curl "https://cmr.earthdata.nasa.gov/search/collections?instrument=1B", simply use pyCMR.searchCollections(instrument="1B") as synonym.

  • For a single result object:

Result is a subclass of python dictionary, so it supports all the functions provided by python dictionary
Result.download(destpath=".")
    Download current result to destpath. Use the current directory as default, can be specified by the user
